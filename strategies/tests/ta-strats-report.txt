## FRAMEWORK BUGS DISCOVERED AND FIXES APPLIED
================================================================================

### Bug 1: Web3.py SignedTransaction Compatibility
- Issue: Web3.py 7.x changed `rawTransaction` attribute to `raw_transaction`
- Fix: Use `getattr(signed_tx, 'raw_transaction', None) or signed_tx.rawTransaction`
- Applied in: All run_anvil.py scripts
- Pattern added to Codebase Patterns section

### Bug 2: LINK Token Not in IntentCompiler Token Registry
- Issue: LINK token address not in TOKEN_ADDRESSES map, causing compilation failures
- Fix: Added LINK to TOKEN_ADDRESSES in almanak/framework/intents/compiler.py:
  ```
  LINK: 0xf97f4df75117a78c1A5a0DBb814Af92458539FB4
  ```
- Commit: 5bca4f5 (US-003)

### Bug 3: Missing RSIData and TokenBalance Imports
- Issue: RSIData and TokenBalance classes not accessible from expected locations
- Fix: Import from `almanak.framework.strategies.intent_strategy`
- Pattern added to Codebase Patterns section

### Bug 4: Lint Errors in run_anvil.py Scripts
- Issue: E402 (module level import not at top) and F841 (unused variable)
- Fix:
  - E402 is expected due to sys.path manipulation (needed for project imports)
  - F841 fixed by removing unused `block` variables
- Pattern: Use `for _ in range()` for unused loop variables (B007)

## HOW TO RUN TESTS
================================================================================

To run individual strategy tests:

1. Set RPC URL in .env:
   ```
   ALMANAK_ARBITRUM_RPC_URL=https://arb-mainnet.g.alchemy.com/v2/YOUR_KEY
   ```

2. Run strategy test:
   ```bash
   python strategies/tests/rsi_reversion/run_anvil.py
   python strategies/tests/macd_crossover/run_anvil.py
   python strategies/tests/ma_crossover/run_anvil.py
   # ... etc
   ```

3. Force specific action:
   ```bash
   python strategies/tests/rsi_reversion/run_anvil.py --action buy
   python strategies/tests/rsi_reversion/run_anvil.py --action sell
   ```
